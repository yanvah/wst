WST is a type definition language that compiles to both TypeScript and Rust from a single source of truth.

Get started Read the docs
$ cargo install --git https://github.com/yanvah/wst
Example

Write it once, get both

Define your types in WST. The compiler produces idiomatic, serde-ready Rust and TypeScript simultaneously.

schema.wst
struct User {
    id    = int64  #required,
    name  = string #required,
    email = string #required,
    role  = Role   #required,
    bio   = string,
}

enum Role {
    Admin,
    Member,
    Guest,
}
schema.ts
export interface User {
  id:    number;
  name:  string;
  email: string;
  role:  Role;
  bio?:  string | null;
}

export enum Role {
  Admin  = "Admin",
  Member = "Member",
  Guest  = "Guest",
}
schema.rs
#[derive(Debug, Clone,
  Serialize, Deserialize)]
pub struct User {
    pub id:    i64,
    pub name:  String,
    pub email: String,
    pub role:  Role,
    #[serde(skip_serializing_if
      = "Option::is_none")]
    pub bio: Option<String>,
}

#[derive(Debug, Clone,
  Serialize, Deserialize)]
pub enum Role {
    Admin, Member, Guest,
}
⚡

Single source of truth

One .wst file generates consistent types for your frontend and backend. No drift, no duplication.

🔒

Compile-time assertions

Assert structural invariants at compile time — like ensuring a struct has a field for every enum case.

🧩

Struct composition

Copy fields between structs, exclude fields with @exclude, and build layered type hierarchies cleanly.

🏷️

Rich metadata tags

Annotate fields with built-in tags like #required and #deprecated, or define your own namespaced tags.

🔗

Imports & namespaces

Split types across files with @import. Use namespace aliases or inline copies to compose schemas.

🎨

VSCode extension

Syntax highlighting and inline error checking via the VSCode Marketplace.

CLI

Simple command-line interface

Compile a file or an entire directory to TypeScript, Rust, or JSON AST.

$ wst -i schema.wst   # validate only
$ wst -i schema.wst -f ts -o schema.ts
$ wst -i schema.wst -f rust -o schema.rs
$ wst -i ./types/ -f ts -o ./generated/   # compile directory
